V2ray WS+Tls协议科学上网
发表于2020-01-24|更新于2020-08-23|网络

关于V2ray

Project V 是一个工具集合，它可以帮助你打造专属的基础通信网络。Project V 的核心工具称为 V2Ray，其主要负责网络协议和功能的实现，与其它 Project V 通信。V2Ray 可以单独运行，也可以和其它工具配合，以提供简便的操作流程。现已全平台支持，在Shadowsocks、ShadowsocksR停止更新的情况下。掌握一个在性能和加密性都比Shadowsocks、ShadowsocksR好的工具是完全有必要的。再加上最近墙又高了，大部分ss和ssr都挂了，v2ray也挂了不少。使用ws+tls呢可以稍微将流量加密一下，让墙不那么容易识别。当然该墙还是要被墙，只是使用时间长短的问题。

开始之前

关于云服务器的申请和域名注册我就不讲了，主要讲一下域名解析和V2ray的安装。
云服务器谷歌的GCP只要有外币的信用卡或者借记卡可以免费试用一年和300美金的赠金
域名注册呢有免费的也有收费的，推荐使用收费的，当然，如果你能弄到免费的更好。

准备工作

  • 一台云服务器，推荐国外的服务器选择离大陆比较近的节点，比如香港。
  • 一个已经注册好的域名
  • 一个能科学上网的环境

域名解析

把你注册的域名解析到你云服务的IP地址上，我用的是cloudflare，提供免费的域名解析服务和cdn加速。

  1. 先进入你注册域名的网站，把域名的域名服务器改为由cloudflare提供的域名服务器
  2. 进入cloudflare官网，没有账号的用邮箱注册登录一下。登录之后，进入首页，点击Add a Site添加一个域名

  1. 在输入框输入你的域名，点击Add site添加

  1. 选择免费的这个套餐，点击Confirm plan 下一步

  1. 添加一条A记录，前面是你网站的前缀，我这里用的是cx，那么我的网站就是cx.csxxx.ml
    后面输入你要解析到的服务器的IP地址，ttl那里随便怎么选，时间越短生效就越快，后面那个小云朵一定要点掉，不使用流量加速服务。
    然后点击Add添加，添加成功后点击con下一步。

  1. 这里也一样继续点con下一步

  1. 到这个页面之后，我们点击上面的这个SSL/TLS设置流量加密

  1. 选择Full完全加密，一定要选择full

  1. 到这里呢域名解析就完了，我们可以ping一下域名，看看是否解析到了我们的服务器上面。

我这里已经解析到我的服务器上面了，也能ping通，证明没问题了。

安装V2ray

通过SSH进入我们的服务器
输入sudo -i获得root权限

输入V2ray一键安装脚本，输入之后回车。（脚本来自233blog）

如果提示 curl: command not found ，那是因为你的 VPS 没装 Curl
ubuntu/debian 系统安装 Curl 方法: apt-get update -y && apt-get install curl -y
centos 系统安装 Curl 方法: yum update -y && yum install curl -y
安装好 curl 之后重新安装脚本

  • 没问题就输入数字1回车安装

  • V2ray传输协议选择 4 WebSocket+Tls

  • 端口这里随便输，直接回车也行，但不能是80和443端口，其他的都可以。

  • 输入刚才已经设置好解析到了服务器上面的那个域名， 一定要正确

  • 这里解析正确的话就输入y。不正确就按ctrl+z撤销，检查域名解析后重新安装。

  • 输入y选择自动配置Tls

  • 默认就行了，直接回车。

  • 广告拦截这里，不需要的可以直接回车，需要的就输入y，我不需要 直接回车了

  • SS这个基本也不大需要，如果想用路由器翻墙，路由器固件不支持V2ray的可以选择配置，不需要就直接回车。

  • 确定没问题就回车安装，有问题就按ctrl+c取消，排查问题之后，重新安装。

  • 到这里安装就算完成了，我们输入V2ray url生成配置链接，导入到V2ray n里面就可以使用了

V2ray N

V2ray N官方下载地址

下载完成后解压，打开v2rayn.exe

在状态栏找到图标右键，选择从剪贴板导入链接

开启http代理 把代理模式改为pac模式

如果不能通过剪贴板导入的，我们可以手动添加vmes服务器，手动输入V2ray的配置信息。

完结

教程写到这里就写完的，如果没问题的话，就可以愉快的开始使用了

遇到问题要善用搜索引擎，大部分坑都已经有人填过了。
